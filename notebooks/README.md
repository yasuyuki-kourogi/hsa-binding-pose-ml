# Notebooks

This folder contains notebooks used in the study workflow.

## Included notebooks

See the list below for each notebook and its role. Before running, update the paths in the **Configuration** (or **User parameters**) cell of each notebook to match your local directory structure.

## Cell structure

Public notebooks are organized, as far as practical, into the following code-cell pattern:

- `Cell 1: Configuration` or `Cell 1: Libraries`
- `Cell 2: Libraries` or `Cell 2: Configuration`
- `Utilities` / `Data loading`
- `Main` / `Execution`
- `Save output` (when applicable)

Workflow notebooks may use additional stepwise sections when that improves readability for interactive use.

**Feature pipeline (run in order):**

1. `01_add_descriptors.ipynb`
   - RDKit 2D descriptor calculation (LigF)
2. `02_add_md_features.ipynb`
   - MD-derived feature extraction from trajectories (MDF; RMSD, contacts, ProLIF, MM/GBSA, etc.)
3. `03_add_pose_rmsd.ipynb`
   - RMSD integration and binary label assignment (label_2p5: ≤2.5 Å, label_5p0: ≤5.0 Å)
4. `04_feature_selection_global.ipynb`
   - Global feature selection (constant/correlated/NZV removal per experiment configuration)
5. `05_lopocv_lgb_optuna.ipynb`
   - LOPOCV + LightGBM + Optuna hyperparameter optimization (50 seeds)

**Feature importance analysis:**

6. `06_feature_importance_shap.ipynb`
   - Global SHAP importance (mean |SHAP| bar plot → Figure 4 A/B; beeswarm → Figure 5)
   - *Note: requires LOPOCV materials generated by notebook 5*
7. `07_feature_importance_permutation.ipynb`
   - Permutation importance (decrease in average precision → Figure 4 C/D)

**External validation (requires final models from notebook 8):**

8. `08_external_validation.ipynb`
   - Final model training (all internal data) + external validation predictions (9UPD nateglinide)
9. `09_shap_local_explanation.ipynb`
   - SHAP local explanation for representative poses (Figure 8)

**Workflow notebooks:**

- `gnina_docking_workflow.ipynb`
   - GNINA docking workflow (10 poses/complex, whole-protein autobox)
- `md_simulation_workflow.ipynb`
   - MD simulation workflow (OpenMM + AMBER ff19SB/GAFF2/TIP3P, 5 ns NPT)

## Curation policy

These notebooks are provided as publication support materials to document the computational logic underlying the analyses, figures, and tables reported in the manuscript.

Computational logic is preserved from the original research notebooks, while release-facing metadata and explanatory markdown have been added to improve readability.

The notebooks are curated representations of the analysis workflow and are not presented as turnkey end-to-end pipelines starting from raw structural sources or a fully standardized external compute environment.
